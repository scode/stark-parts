# Lore Instructions

This directory is for historical artifacts: crawl notes, research reports, source captures, handoffs, and similar material that records what was known at a point in time.

Lore filenames must use this format:

```text
YYYY-MM-DD-title.md
```

Treat lore as historical record, not living project documentation. Do not update lore files as part of routine refactors, terminology cleanup, implementation changes, or spec maintenance. If newer work invalidates or supersedes an older artifact, add a new dated artifact or update the active project docs instead.

Only change existing lore when the user explicitly asks for changes to the lore itself.
